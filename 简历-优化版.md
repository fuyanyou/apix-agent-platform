## 项目经历

### APIX — 企业级多智能体协作平台（独立开发）

**项目描述**：从零构建的全栈 AI Agent 平台，开源项目。支持多智能体并行协作、工具调用、知识库检索、代码安全执行。后端 Python + LangGraph + FastAPI 实现 5 个微服务，前端 Electron + Vue 3 桌面客户端。

#### 模块一：多 Agent 协作运行时（AGENT 服务）

- **LangGraph 图编译引擎**：使用 StateGraph 构建 `agent_node ⇄ tool_node` 循环图，基于 `hash_key = agent_name + config` 做编译缓存，配合 asyncio.Lock 保证并发安全，避免高频 API 调用下的重复编译开销
- **Leader-Worker 异步调度**：Leader 通过工具 `assign_sub_assistant` 将子任务入队 `asyncio.Queue` 后立即返回不阻塞；后台 `sub-agent-worker` 协程消费队列，`asyncio.create_task` 独立执行子 Agent；全部子任务完成后通过事件管道 + 心跳消息唤醒 Leader 汇总结果
- **5 种 Agent 角色**：agent / main_agent / sub_agent / team_leader / team_worker，按角色动态组合工具权限（权限注册表 → 工具列表映射 → 去重），子 Agent 自动禁用越权操作
- **任务状态机**：`submitted → in_progress → completed/failed/cancelled`，状态变更通过 `EventPipe` 发布事件驱动前端 UI 更新和 Leader 唤醒，完成/失败的结果带 TTL（3600s）自动过期清理

#### 模块二：LLM 统一适配与容错

- **4+N 厂商适配**：支持 OpenAI / DeepSeek / MoonShot / Ollama / 自定义，统一 `astream()` 流式接口，按 provider 差异映射参数（如 DeepSeek reasoning 模式）
- **视觉能力自动检测**：内置 base64 极小探针图，首次调用检测多模态支持，结果持久化到 `vision_cache.json`，后续避免重复 API 开销
- **异常分类自愈**：异常通过关键词匹配分三类 — `token_exceed`（触发上下文压缩）、`rate_limit`（退避重试）、`others`（通用提示）。流式中断保留已输出内容，前端接收 `msg_stream_abort` 事件显示中断标记

#### 模块三：RAG 文档问答（FILE 服务）

- **全链路实现**：LibreOffice（Office→PDF）→ PyMuPDF（逐页 JPEG）→ ZIP 打包预览 → 文本切分 → Embedding 向量化 → Milvus 存储 → 语义检索
- **多租户向量隔离**：Milvus 的 `partition_key_field="user_id"` 确保用户间数据隔离，支持不同 provider/model 使用不同向量维度索引
- **索引策略**：当前使用 FLAT（暴力搜索，100% 召回率），架构预留 HNSW 切换接口，适配小数据量阶段的最优选择
- **检索接口**：Agent 通过 `search_knowledge_base` 工具调用，返回 LangChain Document 对象含 metadata（来源文件、页码）

#### 模块四：Electron 桌面客户端（CLIENT）

- 主进程 WebSocket 客户端实现自动重连（指数退避）+ 多订阅者广播，preload 通过 `contextBridge` 安全暴露 70+ IPC 方法
- 四种消息气泡渲染（AI 流式 + 思考面板 / Human / Tool 状态标签 / TODO），13 种流事件驱动前端增量更新
- Pinia 状态管理 + Element Plus UI + Vue Router 路由守卫 + 动态页面注册表（可扩展架构）

#### 模块五：基础设施

- **Docker 安全沙箱**：`SHA256(client_id + work_dir)` 标识容器，bind mount 工作目录，TTL 过期自动清理，定制 Ubuntu 22.04 镜像预装 Python3 + Node.js + Git
- **5 级上下文压缩**：Level 1 丢弃工具输出（可逆）→ Level 2 LLM 摘要压缩（不可逆）→ Level 3 激进丢弃 → Level 4+ 指数截段
- **事件管道**：双通道 — StreamEvent（Agent→前端实时推送）+ CommonEvent（系统内 @on_event 装饰器 + priority 排序 + 信号量限流）
