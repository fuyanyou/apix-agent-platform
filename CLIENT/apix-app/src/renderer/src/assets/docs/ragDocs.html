- · 重要注意事项 · -

文档处理相关
  • 推荐格式：Markdown、TXT、JSON
  • 受限支持：PDF、Word、Excel、PPT
  • 大小限制：单文件不超过 20MB
  • 处理时间：上传后需等待索引完成 (这通常会消耗较长时间)
  • 性能要求：嵌入过程不会使用云端接口，请确保部署文件服务的系统算力充足并安装了 ollama
模型一致性要求
⚠️ 更换嵌入模型后，必须重新处理 Not Indexed 的文档。
  • 原因：向量空间不兼容
  • 操作：点击 Not Indexed 重新生成
查询优化建议
  • 使用完整、明确的自然语言
  • 包含时间、对象、条件等关键信息
  • 系统支持语义匹配与同义词
嵌入模型相关
  • 当前仅支持使用Ollama部署的Embedding模型
  • Ollama目前不提供云端Embedding模型
  • 如何通过Ollama部署请参考 Ollama官方文档 ⎋
数据安全
  • 文档仅用于当前 Agent 系统
  • 系统使用的 LLM 可能由第三方供应商提供
  • 检索结果将会一并发送并可能被供应商缓存
  • 建议定期清理无用文档
  • 敏感内容请提前人工脱敏

- · 常见问题 · -

为什么有时找不到相关信息？
可能原因包括：文档缺失、问题表述偏差或文档尚未处理完成等。
可以上传多少文档？
数量没有硬性限制，但过多文档可能会影响检索效率。
表格和图片能识别吗？
系统主要提取文本内容，复杂表格能力有限，图片识别需要依赖视觉嵌入模型（暂不支持）。
为什么换模型要重传文档？
不同模型生成的向量空间不同，混用会导致检索失真，将会严重影响 Agent 生成回复的质量。
切换嵌入模型后，原有的嵌入结果会不会被删除？
嵌入结果不会被系统自动删除，可以点击文档卡片右上角的删除按钮手动删除。
在哪里获取嵌入模型？
嵌入模型由 ollama 提供，可前往 ollama 官网下载开源模型，不建议在同一台机器上同时下载多个嵌入模型。
