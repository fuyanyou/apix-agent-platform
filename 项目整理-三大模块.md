### 1.3 核心模块详解

#### A. Agent 工厂（AgentCreator）

- **文件**：[agent_creator.py](AGENT/agent_module/apix_agent/apix_agent_core/agent_factory/agent_creator.py)
- **设计模式**：单例 + 工厂模式
- **关键机制**：
  - `_create_agent_core()`：统一构建主/子 Agent 的 LangGraph 图
  - **图编译缓存**：`hash_key = agent_name + 序列化(config)` 作为缓存键，避免高频 LLM 调用下的重复编译开销。缓存结构 `{hash_key: {graph, expire_at, status}}`，asyncio.Lock 保证并发安全
  - **权限收集**：`_collect_permission()` 根据用户配置（纯聊天/文件操作/Web搜索/命令执行等）动态组合工具列表，`forbidden` 权限直接返回空工具集
  - **图结构**：`START → agent_node ⇄ tool_node → END`，工具调用后回到 agent_node 继续思考

#### B. Agent 运行时（AgentRuningtime）

- **文件**：[agent.py](AGENT/agent_module/apix_agent/apix_agent_core/agent.py)
- **核心机制**：后台双协程架构
  - **`sub-agent-worker` 协程**：阻塞消费 `team_task_manager.task_queue`（`asyncio.Queue`），有新任务时 `asyncio.create_task` 启动子 Agent 协程独立执行
  - **`sub-agent-stopper` 协程**：阻塞消费 `stop_request_queue`，收到停止请求后 `task.cancel()` 发送 `CancelledError`，子 Agent 在 `finally` 块标记状态
- **防御性设计**：`start()` 方法可安全多次调用（`if ... is None` 判断），适配 FastAPI 多 worker 模式下 lifespan 可能多次触发的场景

#### C. 五种 Agent 角色体系

| 角色 | 工具权限 | 上下文 | 适用场景 |
|:-----|:--------|:------|:--------|
| `agent` | 全工具（除 assign） | 主对话 | 日常问答、代码编写 |
| `main_agent` | 全工具 + 分配 1 个子Agent | 主对话 | 简单委托任务 |
| `sub_agent` | 受限（无 assign/通信） | 独立上下文 | 被委托执行 |
| `team_leader` | 全工具 + 分配多子Agent | 主对话 | 大型多步骤项目 |
| `team_worker` | 受限工具 | 共享缓存 | 团队协作执行者 |

#### D. 多 Agent 协作调度

- **工具层**：[call_assistant.py](AGENT/agent_module/apix_agent/apix_agent_core/tools/assistant/call_assistant.py)
  - `assign_sub_assistant(agent_identity, task_description, instruction)`：构建 SubAgentState → `team_task_manager.submit_task()` → 入队即返回，Leader 不阻塞
  - `query_sub_assistant(task_ids)`：查询子任务状态/输出/错误
  - `stop_sub_assistant(task_ids)`：发送取消信号
- **调度层**：[team_task_manager.py](AGENT/agent_module/apix_agent/apix_agent_core/agent_task/team_task_manager.py)
  - 任务状态机：`submitted → in_progress → completed/failed/cancelled`
  - 状态存储 `task_state_store: dict[(history_id, task_id), SubAgentState]` + asyncio.Lock
  - TTL 过期清理（默认 3600s）
  - 每个 generation 追踪所有子任务，全部完成后发送 `on_generation_team_task_completed`
- **唤醒机制**：[on_team_task.py](AGENT/agent_module/apix_agent/apix_event_handler/system_handler/on_team_task.py)
  - 监听 `on_generation_team_task_completed` → 检查分支有效性 → 发送心跳消息 → Leader 收到后调用 `query_sub_assistant` 拿结果 → 更新 TODO → 通知用户

#### E. LLM 统一适配层

- **文件**：[llm_adapter.py](AGENT/agent_module/apix_agent/apix_agent_core/LLM/llm_adapter.py)
- **支持厂商**：OpenAI / DeepSeek / MoonShot / Ollama / 小米Mimo / 自定义（4+N 模式）
- **关键设计**：
  - `astream()` 按 provider 差异映射参数（如 DeepSeek 的 reasoning 参数处理）
  - **视觉能力探针**：内置 base64 极小测试图，首次调用检测模型是否支持多模态，结果缓存到 `vision_cache.json`，后续读取文件避免重复 API 开销
  - **异常分类**：`guess_exception_type()` 通过关键词匹配分为 `token_exceed`（触发上下文压缩）、`rate_limit`（退避重试）、`others`（通用错误提示）
  - 流式中断保留已输出内容，通过 `msg_stream_abort` 事件通知前端

#### F. 上下文压缩（5级渐进式）

| Level | 触发条件 | 策略 | 是否可逆 |
|:-----:|:--------|:-----|:-------:|
| 0 | 消息量 < 阈值 | 无操作 | - |
| 1 | 消息量 ≥ 阈值 | 丢弃工具输出（标记 `[Tool Result Outdated]`） | ✅ |
| 2 | Level 1 不够 / token_exceed | **LLM 摘要** 压缩早期消息 → 持久化 shortterm memory | ❌ |
| 3 | Level 2 不够 | 激进丢弃工具消息 (min_keep=2) | ✅ |
| 4+ | Level 3 不够 | 指数截断：`keep = base // 2^(level-3)`，最少保留 2 条 | ✅ |

#### G. 事件管道（双通道架构）

- **Stream Event**（Agent → 前端流式推送）：13 种事件类型，包括 `msg_stream_start/end`、`content_chunk_rtn`、`think_chunk_rtn`、`tool_chunk_rtn`、`tool_exec_start/end`、`msg_stream_abort`、`runtime_warning` 等
- **Common Event**（系统内部事件）：事件注册表 + `@on_event` 装饰器 + priority 排序，信号量限流（Semaphore 100），超时保护
- **阻塞机制**：`send_blocking_event()` 返回 Future，前端 `resolve_block` 后唤醒，用于 `request_user_input` 等需要用户反馈的场景

#### H. Docker 安全沙箱

- **文件**：[agent_sandbox_manager.py](AGENT/agent_module/apix_agent/apix_agent_core/sandbox_manager/agent_sandbox_manager.py)
- **隔离策略**：每个用户+工作区创建独立容器，键 = `SHA256(client_id + work_dir)`
- **容器池化**：TTL 过期自动清理，`_touch()` 续期
- **定制镜像**：基于 Ubuntu 22.04，预装 Python3 + Node.js 20 + Git + build-essential
- **资源绑定**：bind mount 工作目录

#### I. 工具注册体系

- **文件**：[registry.py](AGENT/agent_module/apix_agent/apix_agent_core/tools/registry.py)
- **权限映射**：7 种权限 → 工具列表，支持多权限叠加 + 去重
- **子 Agent 禁用集**：`conflict_tool_set` 防止子 Agent 越权操作
- **20+ 内置工具**：文件 CRUD、Web 搜索（9 个引擎）、知识库检索、代码执行（Bash + Python）、OCR、TODO 管理、Agent 协作等

### 2.3 核心模块详解

#### C. 消息渲染系统

- **消息节点化管理**：任意位置编辑/删除消息 → 自动生成新分支（类似 Git 分支）

#### D. 页面注册表（可扩展架构）

- **文件**：[pageRegistry.js](CLIENT/apix-app/src/renderer/src/router/pageRegistry.js)
- 动态注册机制，新增页面只需在注册表中添加条目，无需修改路由核心代码

## 项目三：FILE — 文件与 RAG 知识库服务

### 3.1 项目定位

APIX 的**文档智能层**。基于 **Python + FastAPI + Milvus + PyMuPDF + LibreOffice**，负责文件上传存储、文档格式转换、向量化嵌入、语义检索。端口 **5094**。

> 注：MEMORY（端口 5093）和 TASK（端口 5090）也是独立微服务，分别负责对话记忆持久化和任务流编排——FILE 是其中与 Agent 工具链耦合最紧密的服务。

### 3.2 核心流程

```

└──────────────┬───────────────────────┘

               ▼
┌──────────────────────────────────────┐

│  ├─ LangChain RecursiveTextSplitter  │
```

### 3.3 关键技术点

#### A. 文档转换管道

- **格式覆盖**：PDF / Word / PPT / Excel / 图片 / 纯文本 / 代码
- **转换链**：`Office格式 → LibreOffice → PDF → PyMuPDF → 逐页JPEG → ZIP预览包`
- **并发控制**：限制 Worker 数为 2，避免 LibreOffice 进程争抢 CPU/内存资源

#### B. 多租户向量隔离

- `partition_key_field="user_id"` 确保每个用户只能检索自己的文档
- 不同 Embedding 模型使用不同向量维度索引，支持按 `provider/model` 过滤
- FLAT 索引保证 100% 召回率（当前数据量不大），架构预留 HNSW 切换空间

#### C. Embedding 工厂

- 统一接口：`get_embed_model(provider, model, api_key)`
- 支持多厂商向量模型动态切换
- 工厂模式 + 懒加载，按需创建避免资源浪费

#### D. 文件记录管理

- [file_record.py](FILE/file_service/routers/file_record.py)：工作区文件 CRUD
- [rag_record.py](FILE/file_service/routers/rag_record.py)：RAG 文档上传/检索/删除
- [skill_record.py](FILE/file_service/routers/skill_record.py)：技能包管理

### 3.4 辅助服务概览

## 数据流全景：从用户输入到 Agent 回复

```
用户输入消息 (Vue assistPage)
       │ window.api.chatComplations()
       ▼
Electron IPC (ipcRenderer.invoke → ipcMain.handle)
       │ ws.send({action: "chat_with_llm", data})
       ▼
AGENT WebSocket (websocket.py)
       │ asyncio.create_task(chat_with_llm)
       ▼
StreamEventHandler.chat_with_llm()
       ├── generation_manager.create_generation()     ── 创建生成记录
       ├── _ensure_config()                           ── 合并/缓存配置
       ├── ai_agent.submit_agent_task()               ── 编译 LangGraph
       │     └── agent_creator.create_agent()
       │           ├── hash 缓存命中? → 直接返回
       │           └── 未命中 → 创建LLM + 收集工具 → compile
       │
       └── agent.astream(initial_state)
             ├── context_prepare   ── 沙箱/历史/技能/文档/记忆
             ├── context_summary   ── 按需 5级压缩
             ├── llm_call          ── Prompt + 流式调用 + 异常重试
             ├── messages_persist  ── 持久化到 MEMORY 服务
             ├── should_continue?
             │     ├── tools → tool_node 执行 → 回到 messages_persist
             │     ├── llm   → 回到 context_summary
             │     └── END   → 结束
             │
             └── 每个节点内 AgentStreamWriter.send_event()
                   │ stream_mode="custom"
                   ▼
             WebSocket → Electron → IPC → Vue 响应式渲染
```
